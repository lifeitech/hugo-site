ChatGPT has taken the world by storm. What's Next?

March 29, 2023 · 12 min · 2480 words · Me | Suggest Changes

In this post, we talk about the techniques behind ChatGPT. We discuss possible applications around ChatGPT, and the impacts that it will bring to society.

On April 11, 2022, I wrote this:

“While current models may still have limited capacity, I’m optimistic that, we should see better and better AI models coming out in the future, and one day machines can have enough intelligence to handle tasks and jobs we do today.”

7 months later, OpenAI released ChatGPT that for the first time shows to the public the real potential of AI. Previously, AI models deployed in the real world are largely passive: face recognition, object detection in autonomous driving cars, recommendation system…they do not have much interaction with users. ChatGPT is different: it is something that everyone can try. Its interactive feedback makes it an immediate success.

As we will see, there is not much “magic” behind ChatGPT. Its building block is attention {% cite attention %} and transformer {% cite transformer %}, which were proposed in 2016 and 2017 respectively, so ChatGPT didn’t employed any radically new technologies. A GPT model is trained on large chunks of Internet to predict the next word. Then it is fine-tuned on prompt-response dataset created by human labelers. Its generated responses to prompts are then evaluated by humans, and the model is further trained to maximize human evaluation score. However, models are one thing, and how to make them work in real world is another. What is marvelous about what OpenAI has done is the exceptional engineering efforts behind data collection, data cleaning, model training, monitoring, and deployments, which could easily require investments of millions. Without such engineering efforts, there won’t be ChatGPT that surprises many of us.

GPT#

GPT {% cite gpt %} is not very complicated. It is a transformer model. All it does is to output a probability distribution over the next word given the input context. To generate next word, one simply selects the word with the highest probability. Here is a snapshot from the GPT paper {% cite gpt %}

{: width=“800” height=“500”} GPT model

Given previous context $[…a]$, the simplest way to generate the next word is to assume that a word only depends on its left neighbor, count the occurence of all words that come after $a$ in the training corpus, and select the word with the highest frequency. This is the n-gram model we talked about earlier.

The Attention model {% cite attention %} is the exact opposite of the Markov assumption of context independence. It is a simple and efficient way to pay attention to previous context when generating the next word. The output $y$ is connected to all previous inputs $(x_1,\ldots,x_n)$ linearly:

$y = \alpha_1 v_1 + \alpha_2 v_2 + \cdots + \alpha_n v_n$,

where the weight $\alpha_i$ represents how much $y$ should pay attention to $x_i$, and $v_i=Vx_i$ with learnable parameter matrix $V$. The context can be very long, for example 3000 tokens or even more. See this post for detail. This way, the computation is easily parallelizable, is very suitable for GPU, and that’s why we can make the model so large by having many of this block.

The Transformer model {% cite transformer %} is attention layer plus residual layer, linear layer and normalization.

{: width=“800” height=“500”} Transformer model

Those extra components are also essential. Residual connections ensures that gradients can directly flow from output to input, and with normalization they results in robust learning. Linear layers add more parameters, and increase expressiveness of the model.

In GPT, there is still fine-tuning on downstream tasks. To use GPT in each type of problem, it has to be fine-tuned on each dataset.

{: width=“800” height=“500”} Fine-tuning objective

Inputs are structured as sequences, with delimiters to separate different parts of input (e.g. context and choices for question answering).

{: width=“800” height=“500”} Fine-tuning input transformations

GPT-2#

The GPT-2 {% cite gpt2 %} model architecture is the same as GPT: Transformers. Here are their main differences:

  1. GPT-2 was trained on a much larger dataset. GPT model was trained on the BooksCorpus {% cite bookcorpus %} dataset. In GPT-2, they scaped the Internet and created a dataset called WebText, which contains 8 million documents for a size of 40 GB.
  2. In GPT-2, layer normalization was moved to the beginning, rather than the end of each sub-block.
  3. An additional layer normalization was added after the final self-attention block.
  4. Context size was increased from 512 to 1024 tokens.

I think those details play a very important roles in improving model performance. But the most significant shift from GPT to GPT-2 is zero-shot. The authors realized that, with enough training data, the model can learn various tasks directly from the dataset (hence the title “language models are unsupervised multitask learners”). Previously, a translation models would be trained on (input language, target language) pairs. A pre-trained generative model has to be fine-tuned on each downstream task with supervised learning. Now this is no longer necessary. First, we can concatenate (input, output) training pairs as sequences, and do generative learning on those sequences. Second, if the model is large enough, and the dataset is large enough so that it contains various tasks, then the model can actually learn to perform such tasks.

{: width=“800” height=“500”} A table in GPT-2 paper showing naturally occuring language translation pairs found in training dataset

And when the model gets larger, it always performs better. Bottlenecks haven’t been seen.

{: width=“800” height=“500”} As model size gets larger, performance gets better, with no bottlenecks observed so far.

GPT-2 is the starting point where we later see amazing performance of AI.

GPT-3#

GPT-3 {% cite gpt3 %} shares basically the same architecture as GPT-2. It has 17B parameters. Here is a short description from the paper:

We use the same model and architecture as GPT-2, including the modified initialization, pre-normalization, and reversible tokenization described therein, with the exception that we use alternating dense and locally banded sparse attention patterns in the layers of the transformer, similar to the Sparse Transformer.

{: width=“800” height=“500”} There is no end in scaling

Different from previous models, they have done extensive experiments, and GPT-3 achieved strong performance across multiple tasks. “Few-shots” in the paper means asking the model to do something with examples included in the context. This is exactly “prompts” that we refer to today.

GPT-3 is already able to write news articles. Back then it was already causing some attention from the public. But still it’s not something that anyone can interact with, so many people were still not aware of the technology, even if it’s really close to what we see as ChatGPT today.

InstructGPT#

Problem with GPT-3: objective is not following instructions.

{: width=“800” height=“500”} InstructGPT

Methods:

  1. GPT-3 is fine-tuned on prompt-desired output dataset created by labelers. Theoretically, if one has a prompt-answering dataset that is large enough and good enough, then there is no need for further steps. However, currently such data labeling is expensive. The fine-tuned model is called SFT (which stands for “supervised fine-tuning”).

  2. Given a prompt, sample several outputs (from $K=4$ to $K=9$) from the model. A human labeler ranks the outputs. A reward model (RM) is trained on human preferences. In {% cite instructGPT %}, the RM model is a GPT-3 model with 6B parameters. The training objective is the pairwise ranking loss

$$ \text{loss}(\theta) = -\frac{1}{K \choose 2} \mathbb{E}{(x, y_w, y_l)\sim D}[\log(\sigma(r\theta(x,y_w) - r_\theta(x, y_l)))] $$

where $r_\theta(x, y)$ is the scalar output of the reward model for prompt $x$ and completion $y$ with parameters $\theta$, $y_w$ is the preferred completion out of the pair of $y_w$ and $y_l$, and $D$ is the dataset of human comparisons.

  1. Fine-tune the SFT model again, such that its output will have a high reward score from the RM model. This is called Proxy Policy Optimization (PPO) in {% cite instructGPT %}, which is about optimizing the following objective:

{: width=“800” height=“500”} InstructGPT

Here, $\pi^{SFT}$ is the fine-tuned model from step 1, which is kept fixed, and $\pi_{\phi}^{RL}$ RL policy is the final chat model we are going to train, which initializes to $\pi^{SFT}$. $x$ is prompts, and $y$ is the response output from the model.

Since the sampling is from the model we are training, after gradient updates, the sample distribution will be different.

The second term (negative KL divergence between distribution $\pi^{RL}(y\mid x)$ and $\pi^{SFT}(y\mid x)$) penalizes deviation from the fine-tuned model from step 1

The third term is the original objective of pre-training (maximizing log likelihood on texts) for GPT-3, here $x$ is training corpus. This is to prevent the model from focusing too much on increasing $r_\theta(x,y)$ and fogetting about next word generation.

Training data size:

The SFT dataset contains about 13k training prompts (from the API and labeler-written), the RM dataset has 33k training prompts (from the API and labeler-written), and the PPO dataset has 31k training prompts (only from the API).

ChatGPT#

ChatGPT is a sibling model to InstructGPT, which is trained to follow an instruction in a prompt and provide a detailed response.

—- OpenAI Blog

{: width=“800” height=“500”} ChatGPT Methods from OpenAI Blog

GPT-4 differs from previous GPT models in that it is multimodal. It accepts both images and texts as inputs and produces text outputs. Little is known about its architecture. Here is the only description from {% cite gpt4 %}:

GPT-4 is a Transformer-style model pre-trained to predict the next token in a document, using both publicly available data (such as internet data) and data licensed from third-party providers. The model was then fine-tuned using Reinforcement Learning from Human Feedback (RLHF). Given both the competitive landscape and the safety implications of large-scale models like GPT-4, this report contains no further details about the architecture (including model size), hardware, training compute, dataset construction, training method, or similar.

— GPT-4 Technical Report, OpenAI

One important thing that OpenAI has been doing is to reliably predict the performance at various scales.

A core component of this project was developing infrastructure and optimization methods that behave predictably across a wide range of scales. This allowed us to accurately predict some aspects of GPT-4’s performance based on models trained with no more than 1/1,000th the compute of GPT-4.

— GPT-4 Technical Report, OpenAI

A large focus of the GPT-4 project was building a deep learning stack that scales predictably. The primary reason is that for very large training runs like GPT-4, it is not feasible to do extensive model-specific tuning. To address this, we developed infrastructure and optimization methods that have very predictable behavior across multiple scales.

— GPT-4 Technical Report, OpenAI

Predicting the future: impacts on society#

OpenAI is very ambitious. It’s ambition is not just limited to coming up with a large NLP model for dialog. Rather, it aims to build an entire ecosystem around ChatGPT, where all future developers and companies have to build products around their models. Regardless or whether it will become the next Google, it is clear that the dynamics have been shifted. The Age of Discovery has began.

Consulting is dead. With ChatGPT Plugins, companies can feed their internal documents to AI models. The models can then answer questions and give suggestions based on knowledge that is much broader than any single human. Special models could also be trained for such consulting services and I’m sure many companies will emerge in the future. The price of such service is surely much cheaper than a consultant’s monthly salary. Many consulting jobs are desktop research, which is basically copy and paste from the Internet. Such jobs do not add much value to the society and are one of those “bulllshit jobs”. ChatGPT and future AI models are probably much better than human at collecting Internet information. Killing those bullshit jobs is good because this means we can stop wasting resources and focus on things are more meaningful.

Apps are dead (?). ChatGPT with plugins has shown us a radical new way of interacting with computers. Rather than installing tons of different apps and using our fingers to give commands to apps, in the future we might just have one unified Chat interface. When we open our phone, there are no longer apps, but we will be greeted with only one chatbot. Whenever we want to do something, we can just ask the chatbot to do it for us.

It’s not that what we have built with apps will become useless, because in many senarios pressing a button is more efficient than talking to a chatbot using natural languages. But I guess the interface stack will shift so that apps will become a lower layer behind the AI interface. I’m also convinced that frontend development is not an inherently hard problem, it is something that can be learned by AI (under massive training).

ChatGPT competitors will be developed. The most advanced AI technology is now at the hands of one giant tech company, the Microsoft. This is of course not acceptable to many. Competition is ongoing, and each tech giant is going to develop their own chatbots. It is hard to say whether in the end we will have many chatbots just like we have many smartphone brands today, or if one company will rule for all. And let’s not forget that, from China’s perspective, the Chinese goverment will surely not allow ChatGPT on their land, just like how they banned Google. China will eventually develop their own ChatGPT alternatives, with strict censorship embedded.

Future development of AI technologies#

More efficient model. Current models are bulky, and costly to train, similar to what computers look like in the 50s and 60s. Maybe we could figure out the exact relationship between data size and model generalization performance, so that we are able to train on minimal data.

Video generation. Current best models can generate texts and images, but models that generate videos haven’t been well-known. This is because video is far more complicated than texts and images, has larger dimension, expensive to train. But I believe we will make fast progress in this direction.

Conclusion#

In this blog post we introduced the technology behind OpenAI’s ChatGPT, and we talked about its potential impacts on society. ChatGPT shows what can be done, and so many similar services will be developed.
